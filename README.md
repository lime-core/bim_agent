# BIM Agent

Клиентский агент для автоматической сборки федеративных BIM-моделей в формате Navisworks (NWD).

Агент устанавливается на рабочей станции с Autodesk Revit и Navisworks, подключается к EIR-серверу и выполняет сборки по запросу из системы. Исходящий трафик только от агента к серверу — никаких входящих подключений не требуется.

## Что делает агент

### Бизнес-назначение

EIR (Engineering Information Requirements) — система управления информационными требованиями к BIM-моделям. Агент — это исполнительная часть системы: он превращает команды из EIR в реальные файлы на диске.

Типичный сценарий:

1. Проектировщики работают в Autodesk Revit, сохраняют модели на Revit Server или в сетевую папку.
2. В EIR настраивается конфигурация сборки: какие разделы (АР, КР, ОВ, ВК...) из каких источников включить.
3. Пользователь нажимает «Запустить сборку» в EIR.
4. Агент автоматически скачивает модели, конвертирует их в NWD и собирает единую федеративную модель.

### Этапы сборки (Build Steps)

| Шаг                | Что происходит                                                                      |
| ------------------ | ----------------------------------------------------------------------------------- |
| `download`         | Скачивает `.rvt` файлы из Revit Server (`RevitServerTool.exe`) или из сетевой папки |
| `convert_rvt_nwd`  | Конвертирует каждый `.rvt` в `.nwd` через `FileToolsTaskRunner.exe` (Navisworks)    |
| `assemble_section` | Собирает все модели одного раздела в один NWD-файл раздела                          |
| `assemble_final`   | Собирает все разделы в итоговую федеративную модель                                 |

### Оптимизация (инкрементальные сборки)

Агент поддерживает кэш NWD-файлов. Если модель не изменилась с прошлой сборки — конвертация пропускается, используется готовый NWD из кэша. Это существенно сокращает время повторных сборок.

### Команды (Agent Commands)

Помимо сборок агент выполняет интерактивные команды из UI:

- **`scan_data_source`** — сканирует Revit Server или сетевую папку, возвращает список `.rvt` файлов. Используется при настройке источников данных в EIR.
- **`test_connection`** — проверяет доступность источника данных.

### Heartbeat

Каждые 30 секунд агент отправляет «пульс» на сервер. По этому сигналу EIR отображает статус агента: `online` / `busy` / `offline`. Если пульс не поступал больше минуты — агент считается недоступным.

## Как это работает

```
┌──────────────────────────────────────────────────────┐
│                  Сеть организации                     │
│                                                      │
│   ┌─────────────┐   HTTP (исходящий)  ┌───────────┐  │
│   │  BIM Agent  │ ──────────────────► │ EIR Сервер│  │
│   │ (Windows PC)│ ◄────────────────── │  (облако) │  │
│   └──────┬──────┘     JSON API        └───────────┘  │
│          │                                           │
│    ┌─────┴──────────────────────┐                    │
│    │  Autodesk Revit Server     │                    │
│    │  или сетевая папка с .rvt  │                    │
│    └────────────────────────────┘                    │
└──────────────────────────────────────────────────────┘
```

**Принцип работы — polling (опрос сервера):**

1. Агент каждые 10 секунд обращается к серверу: _"есть ли новые задания?"_
2. Если задание есть — агент берёт его в работу и выполняет пошагово.
3. После каждого шага агент отправляет прогресс обратно на сервер.
4. Сервер не знает адреса агента и никогда не инициирует соединение первым.

**Итог:** агент может работать за NAT, корпоративным прокси, VPN — любое рабочее место с доступом в интернет (или до сервера) подходит. Не нужно открывать порты или настраивать firewall на стороне клиента.

## Системные требования

| Компонент                  | Требование                                                                 |
| -------------------------- | -------------------------------------------------------------------------- |
| ОС                         | **Windows 10/11 или Windows Server 2019+** (требуется Autodesk)            |
| Node.js                    | **18.0.0 или выше**                                                        |
| Autodesk Navisworks Manage | Версии, соответствующие используемым `.rvt` файлам (2022, 2023, 2024...)   |
| Autodesk Revit             | Только если источники — Revit Server (нужен `RevitServerTool.exe`)         |
| Сеть                       | Доступ к EIR-серверу по HTTP/HTTPS (исходящий)                             |
| Сеть                       | Доступ к Revit Server или сетевым папкам с моделями                        |
| Диск                       | Место под рабочие файлы (зависит от размера моделей; рекомендуется 50+ ГБ) |

> **Важно:** агент должен запускаться под учётной записью Windows, у которой есть доступ к сетевым папкам и лицензия Navisworks Manage.

## Установка и настройка

### 1. Установить Node.js

Перейдите на [nodejs.org](https://nodejs.org/) и скачайте LTS-версию для Windows (18.x или выше).

Проверьте установку в командной строке:

```cmd
node --version
npm --version
```

Должны вывестись номера версий без ошибок.

### 2. Клонировать репозиторий

```cmd
git clone https://github.com/lime-core/bim_agent.git
cd bim_agent
```

Если git не установлен — скачайте архив репозитория через кнопку **Code → Download ZIP** на GitHub, распакуйте в удобное место (например, `C:\bim_agent`).

### 3. Установить зависимости

```cmd
npm install
```

### 4. Создать файл конфигурации

```cmd
copy .env.example .env
```

Откройте `.env` в текстовом редакторе и заполните обязательные поля (см. раздел [Переменные окружения](#переменные-окружения)).

### 5. Получить API-ключ в EIR

1. Войдите в EIR как администратор.
2. Перейдите в раздел **Настройки → Агенты**.
3. Создайте нового агента, скопируйте сгенерированный **API Key**.
4. Вставьте ключ в `.env` → `API_KEY`.

### 6. Проверить подключение

```cmd
npm start
```

При успешном подключении в логе появится:

```
[INFO] Connected to server
[INFO] BIM Agent ready. Waiting for builds...
```

## Переменные окружения

Создайте файл `.env` в корне проекта на основе `.env.example`:

```dotenv
# ─── Обязательные ──────────────────────────────────────
# URL EIR-сервера (без слеша в конце)
SERVER_URL=https://your-eir-server.example.com

# API-ключ агента из раздела "Настройки → Агенты" в EIR UI
API_KEY=your_api_key_here

# ─── Опциональные ──────────────────────────────────────
# Интервал heartbeat в мс (по умолчанию: 30000 = 30 секунд)
# HEARTBEAT_INTERVAL_MS=30000

# Интервал опроса сервера на наличие новых заданий в мс (по умолчанию: 10000 = 10 секунд)
# POLL_INTERVAL_MS=10000

# Рабочая директория для файлов сборки.
# По умолчанию создаётся папка "work" рядом с агентом.
# На Windows рекомендуется указать абсолютный путь:
# WORK_DIR=C:\BIMAgent\work

# Базовый путь установки Autodesk (по умолчанию: C:\Program Files\Autodesk)
# AUTODESK_BASE_PATH=C:\Program Files\Autodesk

# Язык интерфейса Navisworks (по умолчанию: ru-RU)
# NAVISWORKS_LANG=ru-RU

# Таймаут на выполнение одного шага сборки в мс (по умолчанию: 3600000 = 1 час)
# PROCESS_TIMEOUT_MS=3600000
```

### Пути к Autodesk-инструментам

Агент автоматически определяет пути к исполняемым файлам по версии Revit:

- **Navisworks:** `{AUTODESK_BASE_PATH}\Navisworks Manage {version}\FileToolsTaskRunner.exe`
- **Revit Server Tool:** `{AUTODESK_BASE_PATH}\Revit {version}\RevitServerToolCommand\RevitServerTool.exe`

Убедитесь, что нужные версии Autodesk установлены и доступны по стандартным путям.

## Запуск

### Продакшн

```cmd
npm start
```

### Запуск с автообновлением через Git

Если вы хотите получать новые версии агента через `git pull`, используйте режим с отслеживанием файлов — агент автоматически перезапустится после обновления кода:

```cmd
npm run dev
```

### Запуск через bat-файл

В папке агента есть готовый файл `start.bat`. Запускайте двойным кликом. Агент работает, пока открыто окно командной строки.

### Автозапуск при старте Windows (рекомендуется)

В папке агента есть готовый файл `install-autostart.bat`.

Запустите его **от имени администратора** (правой кнопкой → "Запуск от имени администратора") — агент будет автоматически стартовать при каждом включении компьютера.

Чтобы отменить автозапуск, выполните в командной строке от имени администратора:

```bat
schtasks /delete /tn "BIM Agent" /f
```

## Структура рабочих файлов

По умолчанию агент создаёт рабочую директорию `work\` рядом с собой:

```
work\
└── builds\
    └── {buildId}\
        ├── Кэш\
        │   └── {sourceName}\          # Скачанные .rvt файлы (переиспользуются между сборками)
        ├── Сконвертированные\
        │   └── {КОД — Раздел}\        # NWD по разделам
        │       ├── model1.nwd
        │       ├── model2.nwd
        │       └── КОД — Раздел.nwd   # Собранный раздел
        ├── Сборки\
        │   └── {configName}\
        │       └── {configName}.nwd   # Итоговая федеративная модель
        └── Архивы\
            ├── rvt\                   # Архив предыдущих версий .rvt
            └── nwd\                   # Архив предыдущих версий итоговых .nwd
```

Итоговый NWD также записывается в `outputPath`, настраиваемый через EIR UI (раздел **Настройки сборки**).

## Диагностика

### Агент не подключается к серверу

- Проверьте `SERVER_URL` — должен быть доступен с машины агента (попробуйте открыть в браузере).
- Проверьте `API_KEY` — ключ должен быть активен в EIR UI.
- Агент делает 5 попыток подключения с интервалом 5 секунд, после чего завершает работу.

### Ошибка конвертации (FileToolsTaskRunner)

| Код ошибки    | Причина                                                                                               |
| ------------- | ----------------------------------------------------------------------------------------------------- |
| `-2146959355` | Navisworks не запускался на этой машине — откройте вручную, примите лицензионное соглашение, закройте |
| `-2147024891` | Нет прав на запись в рабочую директорию                                                               |
| `-2147024894` | Файл `.rvt` не найден по указанному пути                                                              |

### Агент показывает статус "offline" в EIR

- Убедитесь, что процесс агента запущен (`npm start` или служба Windows).
- Проверьте, что `HEARTBEAT_INTERVAL_MS` не превышает таймаут на сервере (по умолчанию 60 секунд).

### Логи

Агент выводит структурированные логи в stdout с уровнями `[INFO]`, `[WARN]`, `[ERROR]`, `[DEBUG]`. При запуске через NSSM перенаправьте stdout в файл для сохранения истории.
